# NDH Multi‑Domain Atlas Integration Spec v2.5
A governed atlas‑integration architecture enabling multiple NDH domains to be
organized, indexed, visualized, and stabilized within a unified atlas
structure. This spec defines atlas geometry, domain placement rules,
cross‑domain indexing, stability envelopes, lineage anchoring, and
non‑activation guarantees.

---

## 1. Purpose
v2.5 defines the integration of multiple NDH domains into a single atlas.  
The atlas provides:

- spatial‑conceptual organization  
- governed cross‑domain indexing  
- stable multi‑domain visualization  
- lineage‑anchored interpretive boundaries  
- non‑activation guarantees  

This spec completes the NDH v2.x atlas‑integration phase.

---

## 2. Atlas Architecture Overview
The atlas consists of five structural layers:

1. **Atlas Geometry Layer**  
2. **Domain Placement Layer**  
3. **Indexing Layer**  
4. **Stability Layer**  
5. **Governance Layer**  

Each layer enforces strict boundaries to prevent domain bleed, collapse, or
unbounded resonance amplification.
