# NDH Multi‑Domain Atlas Visualization Spec v2.6
A governed visualization architecture for rendering multiple NDH domains
within a unified atlas. This spec defines atlas‑layer rendering rules,
cross‑domain visual boundaries, lineage‑anchored interpretive constraints,
and non‑activation guarantees for multi‑domain cartographic visualization.

---

## 1. Purpose
v2.6 defines how the NDH atlas is visually rendered when multiple domains are
present. It ensures that visualization remains:

- bounded  
- lineage‑anchored  
- non‑coercive  
- emotionally safe  
- non‑activating  

This spec completes the NDH v2.x atlas‑visualization phase.

---

## 2. Visualization Architecture Overview
The atlas visualization system consists of six layers:

1. **Atlas Geometry Layer**  
2. **Domain Rendering Layer**  
3. **Cross‑Domain Overlay Layer**  
4. **Stability Visualization Layer**  
5. **Governance Visualization Layer**  
6. **Interpretive Grid Layer**  

Each layer enforces strict boundaries to prevent domain bleed or collapse.

---

## 3. Atlas Geometry Layer
Defines the spatial‑conceptual frame for visualization.

### 3.1 Geometry Rendering Rules
- render atlas curvature without distortion  
- preserve lineage‑anchored axes  
- maintain non‑activation topology
