### NDH Creativity Engine Applications v1.0  
*How NDH becomes worlds, tools, games, and research engines.*

---

## 1. Purpose

The NDH Creativity Engine is the layer that turns the manifold into **applied experiences**:

- VR worlds  
- cognitive tools  
- adaptive environments  
- narrative simulations  
- games  
- research systems  

All applications must respect:

- NDH API  
- Runtime Engine  
- Accessibility Spec  
- Integration Patterns  
- Tensor Calculus  
- HRD Repository  

---

## 2. Application families

### **2.1 VR Stability Trainer**

- **Goal:** Help users explore and understand stability, basins, margins, and ladder modes in VR.  
- **Core mapping:**  
  - basins → safe zones  
  - corridors → guided paths  
  - margins → warning regions  
  - ladder modes → visual/audio/haptic tone shifts  
- **Accessibility:** audio‑only, haptic‑primary, cognitive‑ease modes.  

---

### **2.2 Cognitive Support / Therapy Tool**

- **Goal:** Provide structured, gentle stability experiences for users needing regulated environments.  
- **Core mapping:**  
  - tensors → internal “weather” indicators  
  - ladder → calm progression  
  - accessibility → strict sensory safety + cognitive‑ease.  
- **Constraints:** HRD‑gated, no diagnostic claims, stability‑only framing.

---

### **2.3 Adaptive Sensory Environment**

- **Goal:** Rooms, apps, or XR spaces that adapt light, sound, and haptics based on NDH state.  
- **Core mapping:**  
  - resonance → ambient sound  
  - stress → intensity reduction triggers  
  - margins → subtle warnings, not alarms.  
- **Accessibility:** profiles drive modality priority (audio/haptic/visual).

---

### **2.4 Narrative Simulation / Story Engine**

- **Goal:** Stories that unfold based on manifold dynamics instead of fixed plot.  
- **Core mapping:**  
  - basins → narrative arcs  
  - corridors → transitions  
  - margins → tension  
  - ladder → emotional regulation.  
- **Output:** games, interactive fiction, experiential films.

---

### **2.5 Game Mechanics / Play Systems**

- **Goal:** Use NDH as a physics engine for play.  
- **Core mapping:**  
  - tensors → forces and buffs/debuffs  
  - basins → safe hubs  
  - margins → high‑risk/high‑reward zones  
  - ladder → difficulty modulation.  
- **Rule:** HRD prevents exploitative or overwhelming designs.

---

### **2.6 Research Engine**

- **Goal:** Use NDH as a structured sandbox for studying stability, perception, and multi‑modal interaction.  
- **Core mapping:**  
  - JDTR → experiment logs  
  - tensors → variables  
  - ladder → intervention model  
  - accessibility → mandatory design constraint.  

---

## 3. Common application pattern

All Creativity Engine applications must:

- **Use NDH API** for state  
- **Run on NDH Runtime Engine**  
- **Respect Accessibility Spec**  
- **Follow Client Integration Patterns**  
- **Pass HRD validation**  
- **Log via JDTR**  

No app is allowed to bypass these layers.
