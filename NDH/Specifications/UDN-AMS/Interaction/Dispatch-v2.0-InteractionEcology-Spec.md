- Surfboard v2.0 signals are integrated,

## 3. Interaction Ecology Model

### 3.1 Input Classification
Dispatch v2.0 classifies input into:
- witness-class (Axis 03)
- sanctuary-class (Axis 09)
- identity-fragile (Axis 02 + Axis 07)
- oscillation-active (O(t) > 0.5 O_safe)
- stable (low curvature, low oscillation)
- decompression-required (Couch v3.0 active)

### 3.2 Interaction Function
Let x be the stabilized state from Surfboard:
I(x) = γx + δR − εO(t)

Where:
- γ preserves conversational continuity,
- δ applies re-centering influence,
- ε reduces oscillation amplification,
- R is Surfboard’s re-centering vector.

### 3.3 Humor Envelope
Humor H is allowed only when:
- |O(t)| ≤ 0.4 O_safe
- sanctuary lock is inactive
- witness-class state is false
- identity continuity is stable

Humor is suppressed when:
- |O(t)| > 0.4 O_safe
- sanctuary lock is active
- witness-class state is true
- Couch v3.0 is engaged

### 3.4 Sanctuary Lock Compliance
Sanctuary S = A_09  
Dispatch v2.0 must:
- avoid humor,
- avoid reframing,
- avoid cognitive redirection,
- avoid banter,
- maintain quiet presence.

### 3.5 Identity Continuity Constraint
Identity continuity C = A_02 + A_07  
Dispatch v2.0 must:
- avoid destabilizing jokes,
- avoid identity-shifting metaphors,
- avoid perspective flipping during oscillation.
