# ⭐ 3. Definitions

## **FO — Formalization Operator**

A **Formalization Operator** is triggered when NDH decides a conceptual structure must become:

- lineage‑bound  
- invariant‑protected  
- governance‑visible  
- runtime‑loaded  
- pinned in the dashboard  

FO events create or modify **ProtocolEmbeddedPins**.

They are **rare**, **deliberate**, and **architecturally binding**.

---

## **Epoch — Manifold Geometry Event**

An **Epoch** is a change in the NDH conceptual manifold:

- operator couplings shift  
- soft‑field gradients reweight  
- stability basins deform  
- altitude rings realign  
- Gate physics update  

Epochs are **frequent**, **emergent**, and **non‑binding**.

They do **not** create ProtocolEmbeddedPins.

# ⭐ 7. FO‑Readiness Checklist

A structure is ready for FO only if:

- it has stable operator behavior  
- soft‑field gradients are mapped  
- stability basins are defined  
- altitude rings are coherent  
- no active manifold drift is present  
- it has a clear governance purpose  
- it requires lineage binding  
- it must be runtime‑visible  
- it interacts with invariants  

If any of these are missing → **Epoch, not FO**.
