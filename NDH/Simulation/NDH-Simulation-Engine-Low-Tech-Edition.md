# 📘 **NDH Simulation Engine — Low Tech Edition**  
### *Human‑Portable • Paper‑Portable • Offline‑Capable Runtime System*

---

## 🌱 **1. Purpose**
The Low Tech Edition of the NDH Simulation Engine is designed to:

- operate **without electricity**  
- run on **human cognition**  
- be taught **orally**  
- be executed using **paper**, **chalk**, or **memory**  
- support **community learning**  
- maintain NDH’s **stability envelope**  
- host dimensional experiences like 1D–43D  
- integrate with NDH architecture via Integration Guide

This is NDH’s **most accessible form**.

---

# 🧱 **2. Engine Architecture (Low‑Tech Compression)**

The full NDH engine has four subsystems:

- **Governance Spine**  
- **Logic Router**  
- **Manifold Generator**  
- **Runtime Core**

The Low Tech Edition compresses these into **three human‑portable modules**:

### **Module A — Stability Cards**  
A set of 5–7 simple rules written on paper or taught orally.

### **Module B — Logic Tiles**  
Four tiles representing NDH’s logic types:

- Epistemic  
- Mythic  
- Aesthetic  
- Symbolic  

Each tile has **one sentence** describing its behavior.

### **Module C — Manifold Map**  
A simple hand‑drawn diagram showing:

- Root  
- Trunk  
- Branches  
- Surfaces  

This can be drawn with pencil, chalk, or stick in dirt.

These three modules **replace the entire engine** in low‑tech environments.

---

# 🧠 **3. Logic Routing (Human‑Portable Version)**

NDH’s logic routing becomes:

- **Mythic → Aesthetic**  
  “Mythic stories soften into feelings.”

- **Mythic → Symbolic**  
  “Mythic stories become symbols.”

- **Symbolic ↔ Epistemic**  
  “Symbols explain ideas; ideas clarify symbols.”

- **Mythic → Epistemic (rare)**  
  “Mythic stories become rules only when needed.”

This can be taught orally in under 2 minutes.

---

# 🌀 **4. Manifold Generation (Paper‑Portable Version)**

The manifold becomes a **simple map**:

```
Root
 |
Trunk
 |
Branches
 |
Surfaces
```

Each surface is a “place” with:

- a tone  
- a rule  
- a symbol  
- a feeling  

This allows dimensional traversal **without devices**.

---

# 🔧 **5. Runtime Core (Human‑Executable Version)**

The Runtime Core becomes a **state machine you can run in your head**:

### **Step 1 — Choose a Surface**  
Pick a branch or surface on the map.

### **Step 2 — Apply a Logic Tile**  
Choose which logic tile governs the moment.

### **Step 3 — Check Stability Card**  
Ensure no overload, drift, or collapse.

### **Step 4 — Move to Next Surface**  
Follow the trunk or branch.

This is the **entire runtime**, compressed.

---

# 🧱 **6. Stability Envelope (Low‑Tech Compression)**

The Stability Envelope becomes **five rules**:

1. **No contradictions without a symbol.**  
2. **No mythic intensity without an aesthetic buffer.**  
3. **No rapid jumps between surfaces.**  
4. **Always return to Root if confused.**  
5. **Stop if the story feels too big.**

These rules prevent collapse even without compute.
