# ⭐ 3. How PNG/SVG Fit Into Your Creative Workflow  
You said you’re limited by poverty — so here’s the optimized strategy:

### 🟦 SVG = the creative backbone  
SVG is:

- free  
- editable in any text editor  
- versionable  
- tiny  
- accessible  
- goat‑union friendly  
- NDH‑geometry compatible  

SVG is the **primary visual format**.

### 🟧 PNG = optional rasterization  
PNG is only used when:

- you need shading  
- you need gradients  
- you need Dark Mode previews  
- you need accessibility brightness floors  

PNG is **secondary**.

### 🟩 ASCII diagrams = zero‑cost fallback  
ASCII diagrams can be served by the Teaching API when:

- SVG is too complex  
- PNG is too expensive  
- visuals are not required  

ASCII is **your poverty‑safe visual engine**.

# ⭐ 4. The Creative Engine Loop  
This is how NDH becomes a creative platform:

### 1. You define a concept  
(e.g., tensor smoothing, goat veto logic, K‑Space slipstream)

### 2. The Teaching API stores it  
as a tile, module, or visual reference.

### 3. The API generates SVG instructions  
(text‑based, free, editable)

### 4. You optionally render PNG  
(using free tools like GIMP or Krita)

### 5. The API serves the content  
to NDH learners, readers, or future you.

This loop is **zero‑cost** and **maximum‑creativity**.
