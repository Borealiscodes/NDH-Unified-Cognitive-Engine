# 🟦 3. Technical Companion  
### *How the layers interact without contamination*

---

## 🟦 3.1 Math Spine → Engine Interface  
The engine uses the spine for:

- dimensional transitions  
- collapse rules  
- invariance checks  
- stability guarantees  
- fixed‑point grounding  

The interface is **one‑way**:

```
Math Spine → Engine
```

The engine **never** writes back into the spine.

---

## 🟧 3.2 Engine → Hyperatlas Interface  
The engine provides:

- state machines  
- progression logic  
- simulation rules  
- dimensional triggers  

The Hyperatlas provides:

- content  
- lore  
- world systems  
- narrative arcs  

This interface is **bidirectional**:

```
Engine ↔ Hyperatlas
```

The Hyperatlas can influence runtime behavior, but **never** the math spine.

---

## 🟣 3.3 Fun Sphere Isolation  
The Fun Sphere is attached to the engine but **never** touches:

- the math spine  
- the Hyperatlas  
- dimensional operators  
- stability proofs  

It is a **sealed module**:

```
Engine → Fun Sphere (one‑way)
```

The Fun Sphere cannot affect the engine.
